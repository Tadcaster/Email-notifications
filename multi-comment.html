Project: New Licensing model v0.4

Manage project notifications

Chapter: User has exceeded number of allowable machines

The idea of self-managing your computers came up today and the idea was that a user could 'self-revoke' a machine that had a license attached to it. This way they do not have to bug sys admins should they upgrade a machine and run out of the 3 instances they are allowed. Can anyone think of why this may not be a good way to take this?

Can the system not 'self-manage' this for the users? Just bump off the first machine from the list once they hit the 4th request for a license? That way we have no UI to build and users do not need to even know about the limits of the system?
